# PROMERA 영상 제작 튜토리얼

이 문서는 PROMERA 시연 영상을 촬영할 때 어떤 데이터를 입력하고, 어떤 버튼을 누르고, 어떤 화면을 보여주면 좋은지 정리한 진행 가이드입니다.

## 0. 촬영 전 준비

### 로컬 실행

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:3000`을 엽니다.

### 깨끗한 시연 상태로 초기화

이전 촬영 데이터가 남아 있으면 질의응답 횟수, 저장 레시피, 저장 정보가 이미 표시될 수 있습니다. 새 시나리오를 찍을 때는 브라우저 개발자 도구 콘솔에서 아래 코드를 실행한 뒤 새로고침하세요.

```js
localStorage.removeItem("promera-user");
localStorage.removeItem("pp-user");
location.reload();
```

## 1. 영상 전체 구성 추천

1. 랜딩 페이지: 흰색 배경 + 바이올렛 브랜드 무드 소개
2. 로그인/온보딩: 초보 소상공인 페르소나 설정
3. 미션 선택: 챗봇 말풍선 안에서 미션 고르기
4. 첫 프롬프트 입력: 일부러 부족하게 입력
5. 재료 체크: 들어간 것/부족한 것 보여주기
6. 한 번에 하나씩 질문: 칩으로 프롬프트 완성
7. 반복 입력 이벤트: 같은 말을 반복했을 때 이벤트 발생
8. Before/After: 최종 프롬프트가 어떻게 좋아졌는지 확인
9. 레시피 모달: 전체 레시피와 완성 프롬프트 확인
10. 좌측 성장 지표: 질의응답 횟수와 성장 변화 확인
11. 저장 레시피 모달: 저장한 레시피를 다시 열어보기
12. Vercel 자동 배포 설정 확인

## 2. 랜딩 페이지 촬영

### 보여줄 포인트

- 흰색 배경 위에 바이올렛 버튼, 보더, 텍스트 강조가 적용되어 있음
- PROMERA 로고와 프롬이 캐릭터가 자연스럽게 노출됨
- 챗봇 미리보기 목업이 “우리 카페 홍보글 써줘” 사례를 보여줌

### 클릭 순서

1. 랜딩 상단의 `무료로 시작하기` 또는 `로그인` 클릭

## 3. 로그인

### 클릭 순서

아래 순서대로 선택합니다.

| 질문 | 선택 |
| --- | --- |

| AI를 주로 어디에 쓰시나요? | `업무·부업에 활용` |

### 목적 상세 입력

카페 홍보에 AI를 써보고 싶어요.

## 6. 챗봇 미션 선택

채팅 화면에 프롬이가 인사하고 `PROMERA Mission` 말풍선이 뜹니다.

### 클릭

`인스타 홍보글 만들기` 클릭

### 보여줄 포인트

- 기존 대화형 챗봇 UI가 유지됨
- 미션 카드가 별도 보드가 아니라 말풍선 안에 표시됨
- 좌측 사이드바에 `첫 질문 연습 · 질의응답 0회`가 표시됨

## 7. 첫 프롬프트 입력

입력창에 아래처럼 일부러 부족하게 입력합니다.

```text
우리 카페 홍보글 써줘
```

전송 버튼을 누르거나 Enter를 누릅니다.

### 예상 화면

`지금 프롬프트에 들어간 것`

- 목적: 신메뉴 홍보글

`아직 부족한 것`

- 카페 위치
- 홍보 대상
- 메뉴/상품 정보
- 플랫폼
- 원하는 말투
- 글자 수

`다음 질문`

```text
먼저 가게가 어디에 있는지 알려주세요.
```

## 8. 반복 입력 이벤트 시연

첫 번째 질문이 나온 상태에서, 사용자가 초보자처럼 같은 말을 다시 입력합니다.

```text
우리 카페 홍보글 써줘
```

### 예상 이벤트

`PROMERA EVENT` 말풍선이 뜹니다.

```text
반복 감지 이벤트
같은 말이 다시 들어왔어요. 이번에는 답을 길게 설명하지 않고, 빠진 재료 하나만 콕 집어서 이어갈게요.
```

### 보여줄 포인트

- 같은 말이 반복되어도 앱이 같은 설명을 계속 반복하지 않음
- 반복 입력은 잘못된 재료값으로 저장되지 않음
- 기존 질문을 유지하면서 다음 행동을 유도함

## 9. 칩으로 재료 채우기

이제 칩을 눌러 프롬프트를 완성합니다.

| 질문 | 누를 칩 |
| --- | --- |
| 먼저 가게가 어디에 있는지 알려주세요. | `부산 해운대` |
| 누구에게 보여줄 홍보글인가요? | `해운대 관광객` |
| 이번에 알리고 싶은 메뉴나 상품은 무엇인가요? | `딸기 크림 라떼` |
| 어디에 올릴 글인가요? | `인스타그램` |
| 어떤 말투로 쓰면 좋을까요? | `밝고 감성적으로` |
| 얼마나 길게 쓰면 좋을까요? | `300자 이내` |

### 중간에 보여줄 포인트

- 좌측 성장 지표의 `질의응답 N회`가 올라감
- `처음 1개 재료`에서 `최고 7/7개`처럼 성장 변화가 보임
- `재료 3개 완성` 같은 이벤트가 중간에 표시될 수 있음
- 각 재료가 `선택 완료`, `직접 입력`, `저장 정보`로 표시됨

## 10. 최종 결과 화면

마지막 칩을 누르면 완성 말풍선이 뜹니다.

### Before

```text
우리 카페 홍보글 써줘
```

### After 예상 출력

```text
부산 해운대에서 운영하는 카페입니다.
해운대 산책 중인 20대 관광객에게 신메뉴 딸기 크림 라떼를 알리는 인스타그램 게시글을 작성해주세요.
말투는 밝고 감성적인 말투로 하고, 전체 길이는 300자 이내로 맞춰주세요.
첫 문장은 방문하고 싶게 시작하고, 메뉴의 매력과 방문 유도 문장을 포함해주세요.
```

### 좋아진 점

- 목적이 추가됐어요
- 위치가 추가됐어요
- 대상이 추가됐어요
- 상품이 추가됐어요
- 플랫폼이 추가됐어요
- 말투가 추가됐어요
- 길이가 추가됐어요

현재 화면에서는 `처음 결과`와 `개선 결과` 미리보기는 표시하지 않습니다. 결과물이 길어질 수 있기 때문에, 핵심은 최종 프롬프트와 좋아진 점에 집중합니다.

## 11. 레시피 모달 시연

### 클릭

`레시피 자세히 보기` 클릭

### 모달에서 보여줄 내용

`레시피 템플릿`

```text
나는 [지역]에서 [가게 종류]를 운영 중이야.
[대상 고객]에게 [상품/이벤트]를 알리는
[플랫폼] 글을 [말투]로 [글자 수] 작성해줘.
```

`완성 프롬프트`

```text
부산 해운대에서 운영하는 카페입니다.
해운대 산책 중인 20대 관광객에게 신메뉴 딸기 크림 라떼를 알리는 인스타그램 게시글을 작성해주세요.
말투는 밝고 감성적인 말투로 하고, 전체 길이는 300자 이내로 맞춰주세요.
첫 문장은 방문하고 싶게 시작하고, 메뉴의 매력과 방문 유도 문장을 포함해주세요.
```

### 클릭

1. `복사` 버튼을 눌러 복사 기능을 보여줍니다.
2. `확인` 또는 X 버튼으로 모달을 닫습니다.
3. `레시피 저장`을 클릭합니다.

## 12. 좌측 저장 레시피 모달 시연

레시피 저장 후 좌측 사이드바의 `내 프롬프트 레시피` 영역에 새 레시피가 표시됩니다.

### 클릭

좌측 저장 레시피의 `전체 보기` 클릭

### 보여줄 포인트

- 저장한 레시피도 모달로 전체 내용 확인 가능
- 템플릿과 완성 프롬프트를 다시 볼 수 있음
- 학습 자산이 쌓이는 서비스처럼 보임

## 13. 질의응답 성장 지표 시연

좌측 `지금 배우는 중` 카드에서 아래 요소를 보여줍니다.

| 항목 | 의미 |
| --- | --- |
| `첫 질문 연습`, `재료 수집가`, `프롬프트 실전러` | 질의응답 횟수 기반 레벨 |
| `질의응답 N회` | 사용자가 프롬이와 실제로 주고받은 횟수 |
| `처음 N개 재료` | 첫 프롬프트에서 잡힌 재료 수 |
| `최고 N/7개` | 가장 많이 채운 재료 수 |
| `처음보다 재료 N개를 더 명확히 넣었어요` | 성장 체감 문구 |
| `완성 프롬프트 N개` | 최종 완성한 프롬프트 수 |
| `저장 레시피 N개` | 저장한 레시피 수 |

레벨은 저장 레시피 수가 아니라 질의응답 횟수로 올라갑니다.

## 14. 개인화 저장 정보 시연

첫 번째 미션에서 위치, 대상, 상품, 플랫폼, 말투를 칩으로 선택하면 이 정보가 저장됩니다.

### 시연 방법

1. `다시 연습` 클릭
2. 입력창에 아래 문장 입력

```text
신메뉴 홍보글 만들어줘
```

### 예상 화면

이전 선택 정보가 `저장 정보`로 자동 체크됩니다.

- 위치: 부산 해운대
- 대상: 해운대 산책 중인 20대 관광객
- 상품: 신메뉴 딸기 크림 라떼
- 플랫폼: 인스타그램
- 말투: 밝고 감성적인 말투

### 보여줄 포인트

- 사용자가 매번 같은 정보를 반복 입력하지 않아도 됨
- PROMERA가 학습자의 맥락을 기억하는 것처럼 보임

## 16. 촬영용 핵심 대사 예시

영상 내레이션으로 아래 흐름을 추천합니다.

```text
AI 초보자는 프롬프트를 어떻게 써야 하는지 모릅니다.
PROMERA는 점수를 매기기보다, 프롬프트에 필요한 재료가 들어갔는지 먼저 보여줍니다.
그리고 한 번에 하나씩만 질문합니다.
사용자는 버튼을 누르거나 짧게 답하면서 프롬프트를 완성합니다.
완성된 프롬프트는 레시피로 저장되고, 다음부터는 내 정보가 자동으로 반영됩니다.
좌측에서는 저장 수가 아니라 실제 질의응답 횟수와 재료 완성도 기준으로 성장합니다.
```

## 17. Vercel 자동 배포 설정

이 저장소에는 아래 파일이 추가되어 있습니다.

- `vercel.json`
- `.github/workflows/vercel-production.yml`

### 방식 A: Vercel Git Integration

Vercel 프로젝트가 GitHub 저장소와 연결되어 있다면, `main` 브랜치가 Production Branch로 설정되어 있을 때 main push마다 자동으로 프로덕션 배포가 생성됩니다.

Vercel 대시보드에서 확인할 것:

1. Project Settings
2. Git
3. Connected Git Repository가 현재 저장소인지 확인
4. Production Branch가 `main`인지 확인
5. Automatic Deployments가 켜져 있는지 확인

### 방식 B: GitHub Actions 배포

`.github/workflows/vercel-production.yml`은 `main` push마다 아래 순서로 실행됩니다.

1. `npm ci`
2. `npm run lint`
3. `vercel pull --yes --environment=production`
4. `vercel build --prod`
5. `vercel deploy --prebuilt --prod`

GitHub 저장소 Settings > Secrets and variables > Actions에 아래 secrets를 등록해야 합니다.

| Secret | 설명 |
| --- | --- |
| `VERCEL_TOKEN` | Vercel Account Settings에서 발급한 토큰 |
| `VERCEL_ORG_ID` | `.vercel/project.json`의 `orgId` 또는 Vercel 프로젝트 정보 |
| `VERCEL_PROJECT_ID` | `.vercel/project.json`의 `projectId` 또는 Vercel 프로젝트 정보 |

로컬에서 프로젝트를 연결하려면:

```bash
npx vercel login
npx vercel link
```

이후 `.vercel/project.json`에서 `orgId`, `projectId`를 확인할 수 있습니다. `.vercel` 폴더는 보통 커밋하지 않습니다.

## 18. 최종 촬영 체크리스트

- 랜딩 배경이 흰색인지 확인
- 바이올렛 버튼/보더/강조색이 일관적인지 확인
- 프롬이 캐릭터가 깨지지 않는지 확인
- 첫 프롬프트에서 부족한 재료가 보이는지 확인
- 반복 입력 이벤트가 뜨는지 확인
- 칩으로 한 단계씩 진행되는지 확인
- 최종 화면에 `처음 결과`, `개선 결과`가 없는지 확인
- `레시피 자세히 보기` 클릭 시 모달이 뜨는지 확인
- `레시피 저장` 후 좌측 `전체 보기` 모달이 뜨는지 확인
- 좌측 질의응답 횟수와 성장 문구가 올라가는지 확인
- 모바일 폭에서도 가로 스크롤이 생기지 않는지 확인
