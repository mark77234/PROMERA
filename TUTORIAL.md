# PROMERA 영상 제작 튜토리얼

이 문서는 Screen Studio로 PROMERA 시연 영상을 촬영할 때 어떤 데이터를 입력하고, 어떤 버튼을 누르고, 어떤 화면을 보여주면 좋은지 정리한 진행 가이드입니다. 모든 화면 문구와 예상 출력은 실제 앱 동작을 검증한 값입니다.

## 0. 촬영 전 준비

### 로컬 실행

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:3000`을 엽니다. 촬영은 데스크톱 해상도(1280px 이상)를 권장합니다. 좌측 사이드바가 보여야 성장 지표를 함께 담을 수 있습니다.

### AI 응답 모드 확인

`.env.local`의 `USE_MOCK_AI` 값이 촬영 결과를 좌우합니다.

- **촬영은 `USE_MOCK_AI=true`(목업)를 권장** — 이 문서의 모든 예상 화면·문구가 결정론적으로 재현됩니다.
- `USE_MOCK_AI=false` + `OPENAI_API_KEY` 설정 시 실제 AI가 재료를 추출·정리합니다. 흐름은 같지만 추출 값 표현이 조금씩 달라질 수 있어 테이크마다 화면이 달라질 수 있습니다. AI 호출이 실패하면 목업으로 대체되지 않고 오류 말풍선이 표시되므로, 촬영 중에는 반드시 목업 모드를 사용하세요.
- 환경변수를 바꾸면 dev 서버를 재시작해야 반영됩니다.

### 깨끗한 시연 상태로 초기화

이전 촬영 데이터가 남아 있으면 질의응답 횟수, 저장 레시피, 저장 정보가 이미 표시될 수 있습니다. 새 시나리오를 찍을 때는 브라우저 개발자 도구 콘솔에서 아래 코드를 실행한 뒤 새로고침하세요.

```js
localStorage.removeItem("promera-user");
localStorage.removeItem("pp-user");
location.reload();
```

## 1. 메인 시나리오 — 페르소나

| 항목 | 설정 |
| --- | --- |
| 이름 | 서연 |
| 나이/직업 | 20대 · 마케팅팀 주니어 직장인 |
| AI 경험 | ChatGPT를 일주일에 몇 번 써봤음 |
| 고민 | 매주 팀장님께 올리는 주간 보고서 작성이 늘 오래 걸림. AI에게 물어봐도 "주간 보고서 써줘"처럼 막연하게 질문해서 뻔한 답만 받음 |
| 목표 | PROMERA로 프롬프트 쓰는 법을 훈련해 업무 문서를 빠르고 정확하게 작성 |

### 스토리 아크 (영상의 감정선)

1. **문제**: AI를 써봤지만 잘 쓴다고 느껴지지 않는 평범한 직장인
2. **발견**: PROMERA는 점수 대신 "빠진 재료"를 보여주고 하나씩만 질문함
3. **성장**: 칩을 눌러가며 프롬프트가 완성되는 경험
4. **자산화**: 완성 프롬프트가 레시피로 저장되고, 다음부터는 내 정보가 자동으로 채워짐
5. **복귀**: 시작 버튼 한 번으로 "다시 오셨네요"와 함께 바로 학습으로 이어짐

## 2. 영상 전체 구성 (씬 리스트)

1. 랜딩 페이지 스크롤 투어
2. 원클릭 시작
3. 온보딩 "처음이신가요?" — 진행 단계 표시
4. 수준 설문 6문항 — 한 문항씩 슬라이드
5. 목적 선택: 업무 문서 작성
6. 상세 목표 입력 (저장 정보 복선)
7. 챗봇 인트로 + 미션 선택
8. 첫 프롬프트 입력 → 재료 체크 (복선 회수: 팀장님 자동 반영)
9. 반복 입력 이벤트
10. 칩으로 재료 채우기 (+ 재료 3개 완성 이벤트)
11. 완성 — Before/After와 좋아진 점
12. 레시피 모달 · 복사 · 저장
13. 좌측 성장 지표
14. 다시 연습 — 저장 정보로 즉시 완성 (개인화 하이라이트)
15. (선택) 시작 화면 → 원클릭 복귀 "다시 오셨네요"
16. (선택) 다른 주제 빠른 탐험

## 3. 씬 1 — 랜딩 페이지

### 보여줄 포인트

- 프롬이 마스코트가 둥실거리는 히어로와 은은한 그라데이션 배경
- 메인 카피: "AI, 배우지 말고 대화하며 익히세요"
- 챗 미리보기 목업이 순차 등장 (사용자 → 프롬이 → 모범 프롬프트)
- 통계 스트립: 학습 주제 6가지 · 실습 미션 12종 · 프롬프트 재료 체크 7가지 · 반복 학습 ∞ (숫자 카운트업)
- 스크롤하며 3단계(STEP 1~3) → 활용 사례 6종(아이·어르신·마케터·직장인·디자이너·개발자) → B2C/B2B/B2G 카드 → 보라 그라데이션 CTA 패널

### 클릭 순서

1. 천천히 최하단까지 스크롤 (섹션별 fade-in 애니메이션이 보이도록)
2. CTA 패널의 `바로 시작하기` 클릭

## 4. 씬 2 — 원클릭 시작

### 클릭

1. `바로 시작하기` 클릭

가입이나 계정 입력 없이 곧바로 온보딩 화면으로 전환됩니다.

## 5. 씬 3 — 온보딩 "처음이신가요?"

상단에 **기본 정보 → 수준 진단 → 학습 주제 → 상세 목표** 4단계 진행 표시가 보입니다. 헤더 좌측의 뒤로가기(‹)도 잠깐 보여주면 UX 완성도가 전달됩니다.

| 항목 | 입력/선택 |
| --- | --- |
| 이름 | `서연` |
| 연령대 | `20대` |
| 직업 | `직장인` |

`다음으로` 클릭

## 6. 씬 4 — 수준 설문 (6문항)

한 문항씩 슬라이드로 전환되고 상단 진행 바가 차오릅니다. 2번 문항부터는 좌측 상단에 `이전 문항으로(←)` 버튼이 생깁니다 — 한 번 눌렀다 돌아오는 컷을 넣으면 좋습니다.

| 질문 | 선택 |
| --- | --- |
| AI를 얼마나 자주 사용하시나요? | `일주일에 몇 번 사용해요` |
| 사용해본 AI 도구 | `ChatGPT` 선택 후 `선택 완료` |
| AI가 원하는 답을 주지 않으면? | `같은 질문을 다시 입력해요` |
| AI에게 질문할 때 어떤 정보까지 넣나요? | `하고 싶은 일만 짧게 말해요` |
| AI 에이전트나 자동화를 써본 적 있나요? | `들어봤지만 써본 적은 없어요` |
| AI를 주로 어디에 쓰시나요? | `업무·부업에 활용` |

페르소나 그대로 — "써보긴 했지만 효율적으로 쓰지 못하는" 답변 조합입니다.

## 7. 씬 5 — 목적 선택

`업무 문서 작성` 카드를 클릭합니다. (카드에 호버하면 화살표가 나타나는 인터랙션을 잠깐 보여주세요.)

## 8. 씬 6 — 상세 목표 입력

아래 문장을 그대로 입력합니다. **"팀장님"이라는 단어가 뒤에서 저장 정보로 자동 반영되는 복선**이므로 꼭 포함하세요.

```text
마케팅팀 주니어인데, 매주 팀장님께 올리는 주간 보고서 작성이 너무 오래 걸려요. AI로 빠르게 잘 쓰고 싶어요.
```

`AI 코치 만나러 가기` 클릭

## 9. 씬 7 — 챗봇 인트로와 미션 선택

프롬이가 타이핑 애니메이션과 함께 인사합니다.

> 서연님, 반가워요. 저는 AI 활용 코치 프롬이예요. … 이제 점수 대신 프롬프트에 필요한 재료를 하나씩 채워볼게요.

이어서 `PROMERA MISSION` 말풍선이 뜹니다.

### 클릭

`업무 보고서 만들기` 클릭

### 보여줄 포인트

- ChatGPT처럼 대화 흐름 안에서 미션 카드가 말풍선으로 표시됨
- 좌측 사이드바: 프로필(서연 · 20대 · 직장인), `첫 질문 연습 · 질의응답 0회`

## 10. 씬 8 — 첫 프롬프트 입력

입력창에 일부러 부족하게 입력하고 전송합니다.

```text
주간 보고서 써줘
```

### 예상 화면 (실측)

`지금 프롬프트에 들어간 것`

- 문서 종류: 주간 보고서 — `입력에서 찾음`
- 읽는 사람: 팀장님 — **`저장 정보`** ← 상세 목표에 적은 "팀장님"이 자동 반영된 것. 여기서 줌인!

`아직 부족한 것`

- 핵심 내용 / 출력 형식 / 문체

`다음 질문`

```text
문서에 꼭 들어가야 할 핵심 내용은 무엇인가요?
```

"왜 이렇게 물어봐야 할까요?" 설명 카드와 답변 칩 4개(성과 3가지 · 일정 조율 · 문제와 해결책 · 다음 액션)가 함께 표시됩니다. 상단 헤더에 **`재료 2/5` 진행 배지**가 나타납니다.

## 11. 씬 9 — 반복 입력 이벤트

초보자처럼 같은 말을 다시 입력합니다.

```text
주간 보고서 써줘
```

### 예상 이벤트

`PROMERA EVENT` 말풍선이 뜹니다.

```text
반복 감지 이벤트
같은 말이 다시 들어왔어요. 이번에는 답을 길게 설명하지 않고, 빠진 재료 하나만 콕 집어서 이어갈게요.
```

같은 질문(핵심 내용)이 유지되고, 반복 입력이 잘못된 재료값으로 저장되지 않는다는 점을 내레이션으로 짚어주세요.

## 12. 씬 10 — 칩으로 재료 채우기

| 질문 | 누를 칩 |
| --- | --- |
| 문서에 꼭 들어가야 할 핵심 내용은 무엇인가요? | `성과 3가지` |
| 어떤 형식으로 정리하면 좋을까요? | `표` |
| 문체는 어떻게 할까요? | `정중하게` |

### 중간에 보여줄 포인트

- `성과 3가지`를 누르면 재료가 3개가 되며 `LEVEL UP — 재료 3개 완성` 이벤트가 뜸: "이제 AI가 대충 쓰는 단계는 지났어요."
- 헤더의 `재료 3/5 → 4/5` 진행 바가 차오름
- 좌측 질의응답 횟수가 계속 올라감 (전송 1 + 반복 1 + 칩 3 = 5회)
- 각 재료에 `선택 완료` 출처 라벨이 붙음

## 13. 씬 11 — 완성 화면

마지막 칩(`정중하게`)을 누르면 완성 말풍선이 뜹니다.

### Before

```text
주간 보고서 써줘
```

### After (실측 출력)

```text
팀장님이 읽을 주간 보고서를 작성해주세요.
핵심 내용은 이번 주 핵심 성과 3가지이고, 표 형식으로 정리해주세요.
문체는 정중한 문체로 해주세요.
```

### 좋아진 점 (5개 모두 표시)

- 문서 종류가 추가됐어요
- 읽는 사람이 추가됐어요
- 핵심 내용이 추가됐어요
- 형식이 추가됐어요
- 문체가 추가됐어요

## 14. 씬 12 — 레시피 모달

1. `레시피 자세히 보기` 클릭 → 모달 표시
2. 모달 내용 확인:

`레시피 템플릿`

```text
[읽는 사람]이 읽을 [문서 종류]를 작성해줘.
핵심 내용은 [주요 내용]이고, [형식]으로 [문체]에 맞춰 정리해줘.
```

3. `복사` 버튼 클릭 (복사 토스트 확인)
4. `확인`으로 모달 닫기
5. `레시피 저장` 클릭 → "프롬프트 레시피를 저장했어요." 토스트

## 15. 씬 13 — 좌측 성장 지표

좌측 사이드바를 줌인해 보여줍니다.

| 항목 | 이 시점 실측값 |
| --- | --- |
| 레벨 | `재료 수집가` (질의응답 3회부터) |
| 질의응답 | 5회 |
| 처음 | 2개 재료 |
| 최고 | 5/5개 |
| 성장 문구 | 처음보다 재료 3개를 더 명확히 넣었어요 |
| 완성 프롬프트 | 1개 · 저장 레시피 1개 |
| 내 프롬프트 레시피 | `업무 보고서 만들기 레시피` + `전체 보기` |

레벨은 저장 수가 아니라 **실제 질의응답 횟수**로 올라간다는 점을 내레이션으로 강조하세요. `전체 보기`를 눌러 저장 레시피 모달도 잠깐 보여줍니다.

## 16. 씬 14 — 다시 연습: 저장 정보로 즉시 완성 (하이라이트)

이 장면이 개인화의 클라이맥스입니다.

1. 완성 말풍선의 `다시 연습` 클릭 — 입력창에 `주간 보고서 써줘`가 자동으로 채워짐
2. 그대로 전송

### 예상 화면

방금 전에는 3번 물어봤던 재료들이 이번에는 **전부 `저장 정보`로 자동 체크**되어, 질문 없이 곧바로 완성 화면이 뜹니다.

- 읽는 사람: 팀장님 — 저장 정보
- 핵심 내용: 이번 주 핵심 성과 3가지 — 저장 정보
- 형식: 표 형식 — 저장 정보
- 문체: 정중한 문체 — 저장 정보

내레이션: "PROMERA는 학습자가 채운 재료를 기억합니다. 같은 정보를 두 번 묻지 않아요."

## 17. 씬 15 — (선택) 원클릭 복귀 경험

1. 좌측 `시작 화면으로` 클릭 → 랜딩으로 복귀
2. `바로 시작하기` 클릭
3. 온보딩·설문을 건너뛰고 바로 챗봇으로 이동하며 토스트가 뜸:

```text
다시 오셨네요, 서연님! 이어서 학습해요 🎉
```

## 18. 씬 16 — (선택) 다른 주제 빠른 탐험

좌측 `새로운 주제로 학습` → 목적 선택으로 이동합니다. 서비스 폭을 보여주고 싶다면 아래 중 하나를 30초 컷으로 추가하세요.

### 홍보 콘텐츠 (소상공인 시나리오)

- 목적: `홍보 콘텐츠 제작` / 상세: `카페 홍보에 AI를 써보고 싶어요.`
- 미션: `인스타 홍보글 만들기` / 첫 입력: `우리 카페 홍보글 써줘`
- 추천 칩: 부산 해운대 → 해운대 관광객 → 딸기 크림 라떼 → 인스타그램 → 밝고 감성적으로 → 300자 이내

### 챗봇 활용

## 20. Vercel 자동 배포 설정

이 저장소에는 아래 파일이 추가되어 있습니다.

- `vercel.json`
- `.github/workflows/vercel-production.yml`

### 방식 A: Vercel Git Integration

Vercel 프로젝트가 GitHub 저장소와 연결되어 있다면, `main` 브랜치가 Production Branch로 설정되어 있을 때 main push마다 자동으로 프로덕션 배포가 생성됩니다.

Vercel 대시보드에서 확인할 것:

1. Project Settings
2. Git
3. Connected Git Repository가 현재 저장소인지 확인
4. Production Branch가 `main`인지 확인
5. Automatic Deployments가 켜져 있는지 확인

### 방식 B: GitHub Actions 배포

`.github/workflows/vercel-production.yml`은 `main` push마다 아래 순서로 실행됩니다.

1. `npm ci`
2. `npm run lint`
3. `vercel pull --yes --environment=production`
4. `vercel build --prod`
5. `vercel deploy --prebuilt --prod`

GitHub 저장소 Settings > Secrets and variables > Actions에 아래 secrets를 등록해야 합니다.

| Secret | 설명 |
| --- | --- |
| `VERCEL_TOKEN` | Vercel Account Settings에서 발급한 토큰 |
| `VERCEL_ORG_ID` | `.vercel/project.json`의 `orgId` 또는 Vercel 프로젝트 정보 |
| `VERCEL_PROJECT_ID` | `.vercel/project.json`의 `projectId` 또는 Vercel 프로젝트 정보 |

로컬에서 프로젝트를 연결하려면:

```bash
npx vercel login
npx vercel link
```

이후 `.vercel/project.json`에서 `orgId`, `projectId`를 확인할 수 있습니다. `.vercel` 폴더는 보통 커밋하지 않습니다.
